# Start Now · MVP PRD（V1 对齐实现版）

> 版本日期：2026-04-09  
> 对齐范围：当前 `01_StartNow` 代码实现（可运行）

---

## 1. 产品目标（MVP）

帮助用户把“大目标”拆成可执行小任务，并通过阶段推进与可视化进度形成持续行动反馈。

MVP 成功标准：
- 用户可完成 1 个完整目标闭环（输入目标 -> 选任务 -> 完成阶段 -> 完成全部）
- 用户可开始下一目标，且历史完成目标可回看
- 页面全流程可运行，无阻塞性错误

---

## 2. 用户流程（当前实现）

1. 进入首页（`/`）
2. 点击「开始行动」进入目标输入（`/goal`）
3. 输入目标后，系统生成工作流（当前为 mock 多阶段）
4. 在「行动地图」（`/path-select`）选择当前阶段未完成任务
5. 在「行动进展」（`/progress`）点击「已完成」推进进度
6. 阶段完成后进入下一阶段；全部完成后可「开始新的目标」
7. 完成目标后归档，用户可在「成长档案」（`/dashboard`）查看历史目标

---

## 3. 页面与功能清单

## 3.1 首页 `Home`（`/`）
- 品牌展示（渐变 Logo）
- Slogan 展示
- 按钮：
  - 「开始行动」 -> `/goal`
  - 「查看演示」 -> `/goal`（演示入口占位）

## 3.2 目标输入 `GoalInput`（`/goal`）
- 标题：你现在最想搞定什么？
- 输入项：
  - 目标（必填，`maxLength=200`）
  - 当前情况（选填，`maxLength=500`）
- 状态：
  - 空输入禁用提交 + 友好提示
  - 加载中禁用输入
  - mock 失败提示文案
- 提交后保存 workflow 到 localStorage

## 3.3 选择第一步 `PathSelect`（`/path-select`）
- 展示当前阶段信息（已完成数/总数）
- 展示当前阶段未完成任务气泡卡片
- 按钮文案动态：
  - 阶段初始：选这个开始
  - 阶段进行中：选这个继续
  - 阶段完成：本阶段已完成（禁用）
- 选择任务后写入 `currentTaskId`，进入 `/progress`

## 3.4 行动进展 `Progress`（`/progress`）
- 左侧：当前任务信息 + 「已完成」
- 右侧：山路进度可视化
  - 虚线路径
  - Lottie 小人沿路径移动（按总进度）
  - 总进度百分比与已完成数
- 阶段控制：
  - 「继续下一步」：当前阶段未完成时可用
  - 「进入下一阶段」：当前阶段完成后高亮可用
- 全部完成后：
  - 「开始新的目标」：归档当前目标后回 `/goal`
  - 「回到首页」
- 行动清单弹层：
  - 按阶段显示全部任务
  - 状态：已完成/当前/未完成

## 3.5 成长档案 `Dashboard`（`/dashboard`）
- 当前进度卡片（当前活动目标）
- 已完成目标列表（从归档读取）：
  - 目标标题
  - 完成任务数/总任务数
  - 阶段数
  - 完成时间
- 「开启新目标」按钮

## 5. 数据结构（localStorage）

## 5.1 当前工作流
- key: `start-now-workflow`
- 类型：`WorkflowState`
- 字段：
  - `stages[]`
  - `currentStageIndex`
  - `currentTaskId`
  - `completedTaskIds[]`
  - `goalTitle`
  - `goalContext`
  - `createdAt`

## 5.2 已完成目标归档
- key: `start-now-goal-history`
- 类型：`GoalArchiveItem[]`
- 字段：
  - `id`
  - `goalTitle`
  - `goalContext`
  - `completedAt`
  - `completedTasks`
  - `totalTasks`
  - `stageCount`

## 6. 技术实现（MVP）

- React 18 + TypeScript + Vite
- Tailwind CSS（自定义主题色）
- React Router v6
- canvas-confetti（完成庆祝）
- lottie-web + 自定义 `LottiePlayer` 封装
- localStorage 持久化
- API：当前 mock（`generateWorkflow`）

---

## 7. 已知限制（MVP 范围内）

- 任务拆解为 mock 模板，未接真实大模型 API
- 登录/账号体系未接入（仅占位按钮）
- 成长档案目前为列表视图，未支持归档详情展开与筛选
- Lottie 资源体积较大，产物包存在体积告警
